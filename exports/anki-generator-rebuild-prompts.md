## Prompt 3 ‚Äî OpenAPI spec + codegen

## Prompt 8 ‚Äî Flashcard generation (text)

> Implement `POST /api/generate` and SSE `POST /api/generate/stream`.
>
> Request body: `{ deckId?, deckName?, parentId?, source: { kind: 'text' | 'pdf', text?, pageTexts? }, customPrompt?, targetCount? }`.
>
> Behavior:
>
> 1. If PDF, use a **page-aware chunker** that packs whole pages into ~6000-char chunks (split oversize pages) and tags each chunk with its starting page. Plain text uses the simple chunker (`pageNumber = null`).
> 2. For each chunk, call the text model with `[YOUR_TEXT_MODEL]` and a system prompt that emphasizes COMPLETE COVERAGE: every fact, definition, mechanism, dose, value, name, criterion must end up on at least one card. The model must also preserve any existing multiple-choice questions verbatim as `cardType: 'mcq'` (stem in `front`, options in `choices`, 0-based `correctIndex`, explanation in `back`).
> 3. Concurrency 3, with proportional card targets and a density floor.
> 4. Stream SSE events with `X-Accel-Buffering: no` and a 12s heartbeat comment so the proxy doesn't buffer or close the stream.
> 5. Insert cards into Postgres ordered by `pageNumber ASC NULLS LAST, createdAt ASC`.
> 6. Return `{ deckId, cardsCreated }`.
>
> Use the retry helper from Prompt 4. Convert provider errors into the strings: `"AI provider quota exceeded..."`, `"AI service temporarily unavailable, please retry"`.

## Prompt 9 ‚Äî Visual flashcard generation from PDF pages

> Extend `POST /api/generate` so when the source is a PDF AND visual mode is enabled, also call `[YOUR_VISION_MODEL]` on PDF page images.
>
> Constants: `VISUAL_BATCH_SIZE = 6`, `VISUAL_CONCURRENCY = 2`, `MAX_VISUAL_BBOX_AREA = 0.55`, `MAX_VISUAL_BBOX_DIM = 0.85`, `CROP_PADDING = 0.04`, `MIN_CROP_DIMENSION = 0.12`, `REGION_OVERLAP_RATIO = 0.25`, `REGION_SNAP_PADDING = 0.025`.
>
> System prompt: "You are an expert visual learning designer. Identify ONLY genuine visual elements: charts, tables, radiology, flowcharts, diagrams, photomicrographs, traces, equations. SKIP pages with no real figure. Return for each: front (question about the figure), back (answer/explanation), figureType, and a TIGHT bbox `{x,y,w,h}` in normalized 0‚Äì1 coords with 3‚Äì5% margin. Forbidden: full-page bboxes, `{0,0,1,1}`."
>
> Pass per-page deterministic hints: the client also sends `pageRegions[]` from PDF.js operator-list parsing (every embedded raster image with normalized coords). Tell the model to point the bbox AT one of these regions, and snap the AI bbox to the nearest detected region with `REGION_OVERLAP_RATIO` minimum overlap. Reject any card whose bbox covers more than `MAX_VISUAL_BBOX_AREA` of the page or spans `MAX_VISUAL_BBOX_DIM` in both dimensions.
>
> Use the `canvas` npm package to crop. Store crops as base64 data URLs in `cards.image`. Visual + text cards from one PDF go into ONE merged deck (no `‚Äì Text` / `‚Äì Visual` split).

## Notes on swapping providers

| Capability | OpenAI-compatible? | Notes |
|---|---|---|
| OpenAI (gpt-4o, gpt-4o-mini, gpt-image-1) | yes | Native; vision built in |
| OpenRouter (any model) | yes | Unified billing, free tiers |
| Anthropic Claude | yes (via Anthropic's OpenAI-compat endpoint) | Vision works |
| Google Gemini (text + vision + `gemini-2.5-flash-image` for gen) | yes (OpenAI-compat) | Cheapest vision tier |
| Groq | yes | Fastest text, no vision/image-gen |
| Together / Fireworks | yes | Many open models |
| Ollama (local) | yes | Free, slower, no image-gen |
| fal.ai (Flux) | no ‚Äî native HTTP | Best price/quality for image-gen |
| Stability AI | no ‚Äî native HTTP | Image-gen only |

Because Prompts 4‚Äì12 always go through `lib/ai-client`, swapping any of the above is a one-file change.
