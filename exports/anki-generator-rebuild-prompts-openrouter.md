# Anki Card Generator — Full Prompt Pack to Rebuild From Scratch (OpenRouter-only)

Use these prompts in order with an AI coding agent (Replit Agent, Cursor, Claude Code, etc.). Every AI capability — text generation, vision (PDF page understanding), and image generation — goes through **OpenRouter** with one API key. Edit only the model names at the top of Prompt 0 if you want to change defaults.

---

## Prompt 0 — Stack & model choices (FILL THIS IN FIRST)

> Lock in the stack before any code. I will refer back to these decisions in every later prompt.
>
> **Single AI provider: OpenRouter** (`https://openrouter.ai/api/v1`, OpenAI-compatible).
>
> One env var only: `OPENROUTER_API_KEY` (get one at <https://openrouter.ai/keys>). Optional override `AI_INTEGRATIONS_OPENAI_BASE_URL` for self-hosted proxies.
>
> Default models (all selectable via env at runtime):
>
> - `AI_TEXT_MODEL` — text flashcards / qbank / explanations. Default: `openai/gpt-oss-120b:free` (free tier, very capable). Other good picks: `anthropic/claude-3.5-sonnet`, `openai/gpt-4o-mini`, `google/gemini-2.0-flash`.
> - `AI_VISION_MODEL` — reads PDF page images for visual cards. Default: `google/gemma-3-27b-it:free`. Other good picks: `openai/gpt-4o`, `anthropic/claude-3.5-sonnet`, `google/gemini-2.0-flash`.
> - `AI_IMAGE_MODEL` — generates illustrations on cards. Default: `google/gemini-2.5-flash-image-preview` (OpenRouter routes to Gemini's image-gen). Other picks if/when listed on OpenRouter: `openai/dall-e-3`, `black-forest-labs/flux-1.1-pro`.
>
> Stack: pnpm monorepo, Node 24, TypeScript 5.9, React 19 + Vite 7 + Tailwind v4 + shadcn/ui (Radix) + wouter + TanStack Query + framer-motion. Backend Express 5 bundled with esbuild. PostgreSQL + Drizzle ORM. OpenAPI 3.1 + Orval codegen for Zod schemas and React Query hooks. Pino structured logging. Deployment: Docker.
>
> Use a single shared `lib/ai-client` so swapping models later is a one-line change.

## Prompt 4 — Shared OpenRouter client

> Create `lib/ai-client/src/client.ts` as a single typed wrapper around OpenRouter.
>
> ```ts
> import OpenAI from "openai";
>
> const apiKey = process.env.OPENROUTER_API_KEY;
> if (!apiKey) throw new Error("OPENROUTER_API_KEY must be set. Get one at https://openrouter.ai/keys");
>
> const baseURL = process.env.AI_INTEGRATIONS_OPENAI_BASE_URL || "https://openrouter.ai/api/v1";
>
> export const openrouter = new OpenAI({
>   apiKey,
>   baseURL,
>   defaultHeaders: {
>     "HTTP-Referer": process.env.OPENROUTER_HTTP_REFERER || "https://anki-generator.local",
>     "X-Title": process.env.OPENROUTER_APP_TITLE || "Anki Card Generator",
>   },
> });
>
> export const TEXT_MODEL   = process.env.AI_TEXT_MODEL   || "openai/gpt-oss-120b:free";
> export const VISION_MODEL = process.env.AI_VISION_MODEL || "google/gemma-3-27b-it:free";
> export const IMAGE_MODEL  = process.env.AI_IMAGE_MODEL  || "google/gemini-2.5-flash-image-preview";
> ```
>
> Add `lib/ai-client/src/text.ts` with `createChatCompletionWithRetry(payload, log, signal?)` that retries on 429/5xx with backoff `[2000, 5000, 10000]` ms and aborts cleanly on `AbortSignal`. Helpers `isRetryableAIError`, `isAbortError`, `getErrorStatus`, `getErrorCode`.
>
> Add `lib/ai-client/src/vision.ts` exporting `callVision({ imageDataUrls, system, user, signal })` that wraps a chat-completions call with content type `image_url` parts (OpenRouter accepts `data:image/...;base64,...` URLs).
>
> Add `lib/ai-client/src/image.ts` exporting `generateImage({ prompt, size = "1024x1024" })`. Implementation: call OpenRouter chat completions with `IMAGE_MODEL` and `modalities: ["image", "text"]`. Parse the response — Gemini-image responses come back as a base64 image part inside `choices[0].message.images[0].image_url.url`. Return `{ imageDataUrl: "data:image/png;base64,..." }`. If the model returns text only (no image), throw `"Image generation failed: model returned no image"`.
>
> Make every client lazy-loaded so missing config returns 503 from routes instead of crashing the server.

## Prompt 8 — Flashcard text generation (OpenRouter text model)

> Implement `POST /api/generate` (sync) and SSE `POST /api/generate/stream`.
>
> Request body: `{ deckId?, deckName?, parentId?, source: { kind: 'text' | 'pdf', text?, pageTexts? }, customPrompt?, targetCount? }`.
>
> Behavior:
>
> 1. If PDF, use a **page-aware chunker** (`buildPagedChunks`) that packs whole pages into ~6000-char chunks (split oversize pages) and tags each chunk with its starting page. Plain text uses the simple chunker (`pageNumber = null`).
> 2. For each chunk, call OpenRouter with `TEXT_MODEL` and this system prompt:
>
>    > You are a meticulous Anki flashcard creator. Your top priority is COMPLETE COVERAGE: every fact, definition, mechanism, classification, dose, value, name, criterion, side-effect, indication, contraindication, formula, step, comparison or relationship in the source text must end up on at least one card. Do not summarise. Do not skip details because they "feel minor". If the text mentions it, the deck must test it.
>    >
>    > Preserve any existing multiple-choice questions verbatim as `cardType: "mcq"` — stem in `front`, options in `choices`, 0-based `correctIndex`, explanation in `back`. For non-MCQ content emit one atomic `cardType: "basic"` card per fact.
>    >
>    > Return ONLY a JSON array. No prose, no markdown fences.
>
>    Append `customPromptBlock(customPrompt)` if provided.
> 3. Concurrency 3, with proportional card targets and a density floor.
> 4. Stream SSE events with `X-Accel-Buffering: no` and a 12s heartbeat comment so the Replit edge / Cloudflare proxy doesn't buffer or close the stream.
> 5. Insert cards into Postgres ordered by `pageNumber ASC NULLS LAST, createdAt ASC`.
> 6. Return `{ deckId, cardsCreated }`.
>
> Use `createChatCompletionWithRetry` from Prompt 4. Convert provider errors to user-friendly strings: `"AI provider quota exceeded. Add credits to your OpenRouter account at openrouter.ai/credits, switch to a free model, or use a different API key."`, `"AI service temporarily unavailable, please retry"`.

## Prompt 9 — Visual flashcard generation (OpenRouter vision model)

> Extend `POST /api/generate` so when the source is a PDF AND visual mode is enabled, also call `VISION_MODEL` on PDF page images.
>
> Constants: `VISUAL_BATCH_SIZE = 6`, `VISUAL_CONCURRENCY = 2`, `MAX_VISUAL_BBOX_AREA = 0.55`, `MAX_VISUAL_BBOX_DIM = 0.85`, `CROP_PADDING = 0.04`, `MIN_CROP_DIMENSION = 0.12`, `REGION_OVERLAP_RATIO = 0.25`, `REGION_SNAP_PADDING = 0.025`.
>
> System prompt (full):
>
> > You are an expert visual learning designer and clinical/scientific illustrator. You convert PDF page images into Anki flashcards centred on the FIGURES shown on each page (NOT on the surrounding prose). You will receive N page image(s).
> >
> > Scan each page and identify ONLY genuine visual elements that have inherent visual content a learner needs to look at. Qualifying categories:
> > 📊 CHARTS — bar, line, pie, scatter, box plots.
> > 📋 TABLES — tabular data: drug doses, classifications, scoring systems, criteria, differential lists, comparison tables, lab values, pharmacology tables.
> > 🩻 RADIOLOGY — X-ray, CT, MRI, ultrasound, ECG strips, traces.
> > 🔄 FLOWCHARTS — algorithms, decision trees, pathways.
> > 🧬 DIAGRAMS — anatomical, mechanism-of-action, cell signaling.
> > 🔬 PHOTOMICROGRAPHS — histology, gross pathology, dermatology photos.
> > 📈 TRACES — ECG, EEG, monitor strips.
> > ƒ EQUATIONS — mathematical / chemical / pharmacokinetic formulas with variables to define.
> >
> > ❌ DO NOT make a card for: pure prose paragraphs, headings/titles alone, page numbers, footnotes, bullet lists of plain text, references, table-of-contents pages, blank pages, copyright notices.
> >
> > For each qualifying figure return: `front` (a question about the figure), `back` (the answer/explanation including caption text if any), `figureType` (one of "chart" | "table" | "radiology" | "flowchart" | "diagram" | "photomicrograph" | "trace" | "equation"), and a TIGHT `bbox: {x, y, w, h}` in normalized 0–1 page coords (top-left origin) with 3–5% margin around the figure + its caption. Forbidden: `{0,0,1,1}` or any near-full-page bbox. Skip pages with no real figure.
> >
> > Return ONLY a JSON array.
>
> Pass per-page deterministic hints in the user message: the client also sends `pageRegions[]` from PDF.js operator-list parsing (every embedded raster image with normalized coords). Format as: `Page N: M embedded image region(s) detected by the PDF parser → [{x,y,w,h}, ...]. STRONGLY PREFER making cards from these regions; the system will snap your bbox to the nearest one.` Tell the model the system will REJECT any visual card whose bbox does not overlap a listed region on a page that has regions.
>
> Server-side: snap the AI bbox to the nearest detected region with `REGION_OVERLAP_RATIO` minimum overlap. Reject any card whose bbox covers more than `MAX_VISUAL_BBOX_AREA` of the page or spans `MAX_VISUAL_BBOX_DIM` in both dimensions. Use the `canvas` npm package to crop with `CROP_PADDING` padding and `MIN_CROP_DIMENSION` minimum. Store crops as base64 data URLs in `cards.image`. Visual + text cards from one PDF go into ONE merged deck (no `– Text` / `– Visual` split).

## Why everything goes through OpenRouter

| Capability | OpenRouter route | Notes |
|---|---|---|
| Text generation | `/chat/completions` with `AI_TEXT_MODEL` | Hundreds of models, free tiers, unified billing |
| Vision (PDF page understanding) | `/chat/completions` with `AI_VISION_MODEL` and `image_url` content parts | Accepts `data:image/...;base64,...` URLs |
| Image generation | `/chat/completions` with `AI_IMAGE_MODEL` and `modalities: ["image","text"]` | Gemini-image and a few others currently supported |
| Streaming | `stream: true` flag (SSE) | Same shape as OpenAI |
| Retries | One shared helper in `lib/ai-client/src/text.ts` | Handles 429 + 5xx with backoff |

One key. One client. One file (`lib/ai-client`) to change when you want a different model.
