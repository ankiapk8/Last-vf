# Anki Card Generator — Full Prompt Pack to Rebuild From Scratch
## (OpenRouter + Neon Postgres edition)

Use these prompts in order with an AI coding agent (Replit Agent, Cursor, Claude Code, etc.). Every AI capability — text, vision, image generation — goes through **OpenRouter**, and the database is **Neon Postgres**. One API key, one connection string.

> ⚠️ **Never paste your `OPENROUTER_API_KEY` or `DATABASE_URL` into the chat or into any file.** Hand them to the agent through its secret-request tool only. If you ever did paste them into chat, treat them as compromised: rotate the OpenRouter key at <https://openrouter.ai/keys> and reset the Neon password from the Neon console. The prompts below are written to **request** the secrets, not to receive them inline.

## Prompt 2 — Database schema (Drizzle + Neon Postgres)

> In `lib/db`, install `drizzle-orm` and `@neondatabase/serverless`. Initialize the driver with the Neon serverless adapter:
>
> ```ts
> import { neon } from "@neondatabase/serverless";
> import { drizzle } from "drizzle-orm/neon-http";
> import * as schema from "./schema";
>
> if (!process.env.DATABASE_URL) {
>   throw new Error("DATABASE_URL must be set. Provide your Neon connection string ending in ?sslmode=require.");
> }
> const sql = neon(process.env.DATABASE_URL);
> export const db = drizzle(sql, { schema });
> ```
>
> Define and export a Drizzle schema with these tables:
>
> - **decks**: `id` (serial PK), `name` (text), `description` (text nullable), `parentId` (FK self-ref, nullable, one level deep), `kind` (`'deck' | 'qbank'`, default `'deck'`), `createdAt`, `updatedAt`.
> - **cards**: `id` (serial PK), `deckId` (FK, on-delete cascade), `front` (text), `back` (text), `tags` (text[]), `image` (text nullable, base64 data URL), `sourceImage` (text nullable), `bbox` (JSON `{x,y,w,h}` nullable), `cardType` (`'basic' | 'mcq'`, default `'basic'`), `choices` (text JSON nullable), `correctIndex` (int nullable), `pageNumber` (int nullable), `createdAt`, `updatedAt`.
> - **generations**: `id`, `deckId`, `kind`, `inputSummary`, `cardCount`, `model`, `error` nullable, `createdAt`.
>
> Export `ensureDatabaseSchema()` that runs idempotent SQL at startup so fresh Neon databases auto-provision. **Critical**: include `CREATE TABLE IF NOT EXISTS` AND `ALTER TABLE … ADD COLUMN IF NOT EXISTS` for **every** column (including `decks.kind` and `cards.page_number`) so existing databases self-heal when the schema grows. Without the ALTERs, an older Neon DB will return 500 on every `/api/decks` request the moment a new column is referenced.
>
> Neon-specific notes:
> - Use the pooled connection string (the one with `-pooler` in the host, default for Neon dashboards) so serverless / Replit Autoscale doesn't exhaust connections.
> - Always include `?sslmode=require`.
> - Neon scale-to-zero may add 200–800 ms cold-start latency on the first query; the health check below should tolerate that.

## Prompt 8 — Flashcard text generation (OpenRouter text model)

> Implement `POST /api/generate` (sync) and SSE `POST /api/generate/stream`.
>
> Request body: `{ deckId?, deckName?, parentId?, source: { kind: 'text' | 'pdf', text?, pageTexts? }, customPrompt?, targetCount? }`.
>
> Behavior:
>
> 1. PDF → page-aware chunker (`buildPagedChunks`) that packs whole pages into ~6000-char chunks (split oversize pages) and tags each chunk with its starting page. Plain text uses the simple chunker (`pageNumber = null`).
> 2. For each chunk, call OpenRouter with `TEXT_MODEL` and this system prompt:
>
>    > You are a meticulous Anki flashcard creator. Your top priority is COMPLETE COVERAGE: every fact, definition, mechanism, classification, dose, value, name, criterion, side-effect, indication, contraindication, formula, step, comparison or relationship in the source text must end up on at least one card. Do not summarise. Do not skip details because they "feel minor". If the text mentions it, the deck must test it.
>    >
>    > Preserve any existing multiple-choice questions verbatim as `cardType: "mcq"` — stem in `front`, options in `choices`, 0-based `correctIndex`, explanation in `back`. For non-MCQ content emit one atomic `cardType: "basic"` card per fact.
>    >
>    > Return ONLY a JSON array. No prose, no markdown fences.
> 3. Concurrency 3, with proportional card targets and a density floor.
> 4. Stream SSE events with `X-Accel-Buffering: no` and a 12s heartbeat comment so the Replit / Cloudflare proxy doesn't buffer or close the stream.
> 5. Insert cards into Postgres ordered by `pageNumber ASC NULLS LAST, createdAt ASC`.
> 6. Return `{ deckId, cardsCreated }`.
>
> Use `createChatCompletionWithRetry` from Prompt 4. Convert provider errors to user-friendly strings: `"AI provider quota exceeded. Add credits to your OpenRouter account at openrouter.ai/credits, switch to a free model, or use a different API key."`, `"AI service temporarily unavailable, please retry"`.

## Prompt 9 — Visual flashcard generation (OpenRouter vision model)

> Extend `POST /api/generate` so when the source is a PDF AND visual mode is enabled, also call `VISION_MODEL` on PDF page images.
>
> Constants: `VISUAL_BATCH_SIZE = 6`, `VISUAL_CONCURRENCY = 2`, `MAX_VISUAL_BBOX_AREA = 0.55`, `MAX_VISUAL_BBOX_DIM = 0.85`, `CROP_PADDING = 0.04`, `MIN_CROP_DIMENSION = 0.12`, `REGION_OVERLAP_RATIO = 0.25`, `REGION_SNAP_PADDING = 0.025`.
>
> System prompt (full):
>
> > You are an expert visual learning designer and clinical/scientific illustrator. You convert PDF page images into Anki flashcards centred on the FIGURES shown on each page (NOT on the surrounding prose). You will receive N page image(s).
> >
> > Identify ONLY genuine visual elements that have inherent visual content a learner needs to look at: 📊 charts, 📋 tables (drug doses, classifications, scoring systems, criteria, lab values), 🩻 radiology / ECG strips, 🔄 flowcharts / decision trees, 🧬 anatomical or mechanism diagrams, 🔬 photomicrographs, 📈 traces, ƒ typeset equations.
> >
> > ❌ DO NOT make a card for: pure prose paragraphs, headings/titles alone, page numbers, footnotes, bullet lists of plain text, references, table-of-contents pages, blank pages, copyright notices.
> >
> > For each qualifying figure return: `front` (a question about the figure), `back` (the answer/explanation including caption text if any), `figureType`, and a TIGHT `bbox: {x, y, w, h}` in normalized 0–1 page coords (top-left origin) with 3–5% margin around the figure + its caption. Forbidden: `{0,0,1,1}` or any near-full-page bbox. Skip pages with no real figure.
> >
> > Return ONLY a JSON array.
>
> Pass per-page deterministic hints in the user message: `pageRegions[]` from PDF.js operator-list parsing (every embedded raster image with normalized coords). Phrase the hints as a STRONG PREFERENCE, not a hard rule: when a listed region matches a real figure, point the bbox at it; if a page contains a vector chart, table, equation, or scanned-as-page diagram with no listed region, the model may STILL emit a card with a tight bbox.
>
> Server-side: try to snap the AI bbox to the nearest detected region with `REGION_OVERLAP_RATIO` minimum overlap. **If snap fails, KEEP the AI bbox** instead of dropping the card — the size/area limits already reject obvious failures, and silently dropping cards on vector pages was a real bug in the original. Reject any card whose bbox covers more than `MAX_VISUAL_BBOX_AREA` of the page or spans `MAX_VISUAL_BBOX_DIM` in both dimensions. Use the `canvas` npm package to crop with `CROP_PADDING` padding and `MIN_CROP_DIMENSION` minimum. Store crops as base64 data URLs in `cards.image`.
>
> Before inserting visual cards, **filter out any whose `image` is empty or non-string** — the AI sometimes returns a card without an image and `c.image.startsWith(...)` will crash the whole insert.
>
> Visual + text cards from one PDF go into ONE merged deck (no `– Text` / `– Visual` split).

## Common pitfalls (from the original build) — avoid these

1. **Schema drift on existing databases.** Always pair `CREATE TABLE IF NOT EXISTS` with `ALTER TABLE … ADD COLUMN IF NOT EXISTS` for **every** column. The original build silently 500'd every `/api/decks` request after `kind` was added because the migration only handled fresh databases.
2. **Silently dropping visual cards.** Don't reject AI bboxes that miss PDF.js-detected raster regions — vector charts, tables, and scanned-page PDFs have no detected regions but still contain real figures.
3. **`c.image.startsWith(...)` insert crash.** Filter out visual cards with empty/null `image` before insert.
4. **MCQ rendering crash.** Bounds-check `correctIndex` against `choices.length` before reading `choices[correctIndex]`.
5. **Tabs only when both types exist.** Always show All / Text / Visual tabs on regular decks so users can see the structure even when one side is empty.
6. **Never paste secrets into chat or files.** Use the agent's secret-request tool. If a secret leaks, rotate it immediately.

## Why everything goes through OpenRouter + Neon

| Concern | Pick | Notes |
|---|---|---|
| Text generation | OpenRouter `/chat/completions` with `AI_TEXT_MODEL` | Hundreds of models, free tiers, unified billing |
| Vision (PDF page understanding) | OpenRouter `/chat/completions` with `AI_VISION_MODEL` and `image_url` parts | Accepts `data:image/...;base64,...` URLs |
| Image generation | OpenRouter `/chat/completions` with `AI_IMAGE_MODEL` and `modalities: ["image","text"]` | Gemini-image and a few others |
| Streaming | `stream: true` (SSE) | Same shape as OpenAI |
| Database | Neon Postgres + `@neondatabase/serverless` driver | Scale-to-zero, branching, generous free tier |
| Migrations | `ensureDatabaseSchema()` at boot — `CREATE TABLE IF NOT EXISTS` + `ALTER TABLE … ADD COLUMN IF NOT EXISTS` | Fresh and existing Neon DBs both self-heal |

One AI key. One DB URL. One file (`lib/ai-client`) to change when you want a different model. One file (`lib/db/src/index.ts`) to change when you want a different database.
